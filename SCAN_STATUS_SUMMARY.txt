================================================================================
UNDERSTANDING SCAN METRICS
================================================================================

PAGES DISCOVERED:
- Total unique URLs found during crawl
- Includes all links extracted from pages
- Example: 63 URLs discovered

PAGES CRAWLED:
- URLs successfully fetched and processed
- Only counts successful HTTP responses
- Example: 14 pages crawled

WHY DIFFERENT?
- Depth limiting (stops at max_depth)
- Scope filtering (out-of-scope URLs)
- Error handling (timeouts, 404s)
- Content type filtering (non-HTML)
- Robots.txt restrictions

This is NORMAL and EXPECTED behavior!

================================================================================
TROUBLESHOOTING
================================================================================

ISSUE: Scan takes too long
SOLUTION: Use depth 1 and reduce concurrency
  python -m src.cli.commands scan <URL> --no-warning -d 1 -c 2

ISSUE: Connection timeouts
SOLUTION: Increase timeout value
  python -m src.cli.commands scan <URL> --no-warning --timeout 30

ISSUE: Too many requests
SOLUTION: Increase delays
  python -m src.cli.commands scan <URL> --no-warning --delay-min 0.5 --delay-max 1.0
