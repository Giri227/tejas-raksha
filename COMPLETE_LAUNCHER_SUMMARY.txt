================================================================================
WHAT HAPPENS BEHIND THE SCENES
================================================================================

When you run the launcher:

1. INITIALIZATION:
   - Displays banner
   - Activates virtual environment (PS1/BAT)
   - Checks if venv exists

2. USER INPUT:
   - Prompts for target URL
   - Validates URL is not empty
   - Prompts for scan depth (1/2/3)
   - Maps choice to depth value
   - Prompts for report format (1/2/3)
   - Maps choice to format flags

3. CONFIGURATION:
   - Generates timestamp
   - Creates output directory name
   - Displays configuration summary
   - Asks for confirmation

4. EXECUTION:
   - Builds command with all parameters
   - Executes: python -m src.cli.commands scan <URL> --no-warning -d <DEPTH> -o <DIR> -f <FORMATS> -v
   - Shows progress and output
   - Captures exit code

5. COMPLETION:
   - Checks if scan succeeded
   - Finds HTML report file
   - Opens report in default browser
   - Displays success/failure message
   - Waits for user to press Enter

================================================================================
TROUBLESHOOTING
================================================================================

ISSUE: "Cannot be loaded because running scripts is disabled"
SOLUTION: Run PowerShell as Administrator:
  Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

ISSUE: "Virtual environment not found"
SOLUTION: Edit the venv path in the script to match your installation

ISSUE: Script doesn't start
SOLUTION: Make sure you're in the agri-scanner folder:
  cd E:\TejasRaksha\agri-scanner

ISSUE: Python script requires venv activation
SOLUTION: This is expected. Activate venv first:
  & e:/TejasRaksha/.venv/Scripts/Activate.ps1

ISSUE: Report doesn't open automatically
SOLUTION: Manually open from the output folder:
  Invoke-Item ./scan-results-*/scan_report_*.html
