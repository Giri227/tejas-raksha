HOW TO RUN
================================================================================

DEVELOPMENT:
------------
1. Start Redis:
   redis-server

2. Start Celery Worker:
   cd agri-scanner/dashboard/backend
   venv\Scripts\activate
   python run_celery.py

3. Start FastAPI Server:
   cd agri-scanner/dashboard/backend
   venv\Scripts\activate
   python run_server.py

4. Access Dashboard:
   http://127.0.0.1:8000

PRODUCTION:
-----------
See DEPLOYMENT_GUIDE.md for:
- PostgreSQL setup
- Gunicorn configuration
- Systemd services
- Nginx reverse proxy
- SSL with Let's Encrypt
- Docker deployment
- Monitoring with Flower

API ENDPOINTS
================================================================================

Authentication:
- POST   /api/auth/register       # Register user
- POST   /api/auth/login          # Login
- POST   /api/auth/logout         # Logout
- GET    /api/auth/me             # Current user

Scans:
- GET    /api/scans               # List scans
- POST   /api/scans               # Create scan
- GET    /api/scans/{id}          # Get scan
- DELETE /api/scans/{id}          # Delete scan
- POST   /api/scans/{id}/pause    # Pause scan
- POST   /api/scans/{id}/resume   # Resume scan
- POST   /api/scans/{id}/stop     # Stop scan
- GET    /api/scans/{id}/progress # Get progress
- GET    /api/scans/{id}/findings # Get findings
- GET    /api/scans/statistics/summary  # Statistics
- PATCH  /api/scans/findings/{id} # Update finding

Pages:
- GET    /                        # Dashboard
- GET    /scans                   # Scans list
- GET    /scans/new               # New scan form
- GET    /scans/{id}              # Scan details
- GET    /scans/{id}/live         # Live monitoring
- GET    /health                  # Health check
- GET    /docs                    # API documentation

WebSocket:
- WS     /ws                      # WebSocket endpoint
